# 03. 인물 매칭 시스템

## 1. 개요
동일한 인물이 여러 은행/계좌에서 다른 이름(예: "홍길동", "홍 길동", "KANG")으로 찍히는 문제를 해결하기 위한 시스템.

## 2. 인물(Person)과 별칭(Alias) 구조
- `Person`: 대표 이름
- `PersonAlias`: 매칭될 수 있는 여러 예금자명 문자열들

## 3. 합치기(Merge) 프로세스
1. 사용자가 여러 인물을 선택.
2. 타겟(Target) 인물을 지정.
3. 소스(Source) 인물들의 모든 `PersonAlias`를 타겟으로 이전.
4. 기존 소스에 연결된 모든 거래내역(`Transaction`)의 `person_id`를 타겟 ID로 업데이트.
5. 병합 후 빈 소스 인물 데이터는 삭제.

## 4. 분리(Split) 프로세스
- 특정 별칭이 잘못 묶인 경우, 해당 별칭을 새로운 `Person`으로 생성.
- 해당 별칭으로 기록된 거래내역들을 새로 생성된 인물로 재할당.
